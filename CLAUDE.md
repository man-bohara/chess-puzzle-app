# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository layout

Monorepo with three top-level workspaces:

- **`mobile/`** ‚Äî Flutter app for players. Solves puzzles, tracks per-user progress.
- **`admin/`** ‚Äî Next.js 16 + React 19 + Tailwind v4 admin portal for authoring puzzles.
- **`firebase/`** ‚Äî Shared Firebase config: Firestore security rules, indexes, emulator setup. Deployed once; both clients talk to the same project.

The two clients are **not** linked by a shared package ‚Äî they share a contract only through Firestore documents (see "Data model" below). When you change the puzzle schema, update both `mobile/lib/features/puzzles/domain/puzzle.dart` and `admin/lib/puzzles.ts` together.

## High-level architecture

### Firestore data model

```
/puzzles/{puzzleId}
  fen          string       starting position (FEN)
  solution     string[]     UCI moves; includes opponent replies (mobile auto-plays them)
  difficulty   number       1‚Äì5
  themes       string[]     e.g. ["fork", "mateIn2"]
  published    boolean      drafts (false) are admin-only-visible
  createdAt    timestamp
  createdBy    string       admin uid

/users/{uid}/progress/{puzzleId}
  solved       boolean
  attempts     number       FieldValue.increment(1) per attempt
  solvedAt     timestamp
```

Access is gated by `firebase/firestore.rules`:
- `/puzzles` is **world-readable**, **admin-only-writable** (requires custom claim `admin: true`).
- `/users/{uid}/**` is only readable/writable by `uid` itself.

Granting admin: set the custom claim via Firebase Admin SDK or a Cloud Function. The admin portal does not bootstrap this ‚Äî provision the first admin manually.

### Mobile (`mobile/lib/`)

Feature-first layout. Each feature has `data/` (Firebase) ‚Üí `application/` (state) ‚Üí `presentation/` (UI), with `domain/` for plain models.

- `main.dart` initializes Firebase + wraps app in `ProviderScope` (Riverpod).
- `app.dart` builds `MaterialApp.router` from `appRouterProvider`.
- `core/router/app_router.dart` ‚Äî go_router with auth-aware redirect: unauthenticated users bounce to `/login`. Routes: `/login`, `/` (puzzle list), `/puzzles/:id` (solver).
- `features/auth/data/auth_repository.dart` ‚Äî exposes `firebaseAuthProvider`, `authStateProvider` (the stream the router watches), `authRepositoryProvider`.
- `features/puzzles/data/puzzle_repository.dart` ‚Äî `publishedPuzzlesProvider` (stream) and `puzzleByIdProvider(id)` (future). All reads filter `published == true`.
- `features/puzzles/application/puzzle_controller.dart` ‚Äî `StateNotifier` per puzzle attempt. `tryMove(uci)` validates against `puzzle.solution[moveIndex]`; on success it also plays the canned opponent reply at `solution[moveIndex+1]`. Wrong move ‚Üí `PuzzleStatus.failed`.
- `features/puzzles/presentation/puzzle_solver_screen.dart` ‚Äî board rendering is **not** wired up; the file has a `Placeholder` and a TODO pointing to where to plug in your board-renderer package of choice. The chess engine logic (legality, FEN) is done via the `chess` package.
- `features/progress/data/progress_repository.dart` ‚Äî writes to `/users/{uid}/progress/{puzzleId}` after each attempt.

`lib/firebase_options.dart` is a **placeholder** that throws at runtime. The real file is generated by `flutterfire configure`; it is also in `.gitignore` because it contains per-platform API keys.

### Admin (`admin/`)

Next.js 16 App Router (TypeScript) with Tailwind v4 (CSS-first config in `app/globals.css`) and shadcn/ui (slate, default style, in `components/ui/`). Path alias `@/*` ‚Üí repo root.

- `app/layout.tsx` ‚Äî root layout with top-nav (Dashboard / Puzzles / New / Sign in).
- `app/page.tsx` ‚Äî dashboard landing.
- `app/login/page.tsx` ‚Äî client component, email/password sign-in via Firebase Auth.
- `app/puzzles/page.tsx` ‚Äî list of all puzzles (drafts + published) in a shadcn `Table`.
- `app/puzzles/new/page.tsx` ‚Äî create form. Splits the solution textarea on whitespace into UCI moves; themes split on commas.
- `lib/firebase.ts` ‚Äî lazy-init helpers `getFirebaseApp()`, `getFirebaseAuth()`, `getFirebaseDb()`. Reads `NEXT_PUBLIC_FIREBASE_*` env vars.
- `lib/puzzles.ts` ‚Äî `listAllPuzzles()` and `createPuzzle()`. `createPuzzle()` stamps `createdBy` from the current auth user; Firestore rules then check the `admin: true` claim.

There is **no auth middleware yet** ‚Äî pages render even when signed-out, but Firestore writes will be rejected by rules. If you add a protected route, gate it client-side on `onAuthStateChanged` or wire up a server-side session cookie.

#### Important: Next.js 16

This is Next.js 16, **not** 14/15. APIs and conventions differ from older training data. Before writing non-trivial Next.js code, consult `admin/AGENTS.md` and the bundled docs at `admin/node_modules/next/dist/docs/`. The project's own `admin/CLAUDE.md` imports `AGENTS.md` for this reason.

Tailwind here is v4 (CSS-first): there is **no** `tailwind.config.ts`. Theme tokens live in `app/globals.css` under `@theme inline { ... }`.

## Setup checklist

When starting from a fresh clone:

1. `cd mobile && flutter pub get`
2. `cd admin && npm install`
3. Create Firebase project, enable Auth (Email/Password) + Firestore.
4. `cd mobile && flutterfire configure` (generates real `firebase_options.dart`).
5. `cp admin/.env.local.example admin/.env.local`, fill in web SDK config from Firebase Console.
6. `cp firebase/.firebaserc.example firebase/.firebaserc`, set `projects.default`.
7. `cd firebase && firebase deploy --only firestore:rules,firestore:indexes`.
8. Grant yourself the `admin: true` custom claim (one-off, via Admin SDK).

## Conventions

- **UCI everywhere** ‚Äî solution moves are stored and exchanged as UCI strings (`e2e4`, `a7a8q`). Both clients assume this; do not introduce SAN/PGN at the storage layer.
- **`solution` includes opponent replies** ‚Äî the player only chooses on odd indices (0, 2, ‚Ä¶); even indices are auto-played by the mobile controller. The admin form's helper text reflects this ‚Äî keep it in sync if you change semantics.
- **Don't commit `firebase_options.dart`, `.env.local`, `.firebaserc`, or `google-services.json` / `GoogleService-Info.plist`** ‚Äî already in `.gitignore`.
